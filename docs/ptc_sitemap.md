## Part 2: Issues with Current Structure

| Issue | Impact |
|-------|--------|
| **Heavy duplication** вЂ” Financial Aid, Admissions, Testing, and Career Center pages are copy-pasted across both campus sites and the main site | Maintenance burden; inconsistent updates; user confusion |
| **No centralized programs directory** вЂ” Main site has no programs listing; users must know which campus to visit | Poor discovery; lost prospective students |
| **Missing COE compliance pages** вЂ” Neither campus publishes CPL (Completion, Placement, Licensure) data, Gainful Employment disclosures, or campus-specific consumer information | Accreditation risk |
| **Inconsistent nav structure** вЂ” Main site has different categories than campus sites | Disorienting cross-site navigation |
| **Orphaned/dead-end content** вЂ” Some pages only accessible via sitemap, not nav | Content is effectively invisible |
| **No shared global nav** вЂ” Switching between main site and campus sites feels like visiting unrelated websites | Brand fragmentation |

---

## Part 3: Proposed Restructured Sitemap

> [!IMPORTANT]
> **Guiding Principle:** The main site is the **portal** for institutional information and program discovery. Campus sites are **standalone hubs** with campus-specific programs, CPL data, and accreditation info (required by COE). Shared policies and resources live on the main site and are linked вЂ” not duplicated вЂ” from campus sites.
