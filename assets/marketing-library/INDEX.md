## Workflow when we pull an asset into the repo

1. **Pick the page need first** (e.g., "homepage hero video" or "Diesel program page hero photo"). Don't pre-curate without a destination.
2. **Copy single file or small folder** from F: into a temp working location (e.g., `assets/_unprocessed/` — gitignored).
3. **Optimize for web:**
   - **Photos:** Resize longest edge to 1600–2000px (hero) or 800–1200px (cards/thumbs). Save as JPG quality 80% or WebP. Target 150–500 KB per image.
   - **Hero videos:** Re-encode to web-friendly `.mp4` (H.264) at 1080p or 720p, target ≤10 MB for autoplay backgrounds, ≤30 MB for embedded clips. Strip audio if used as muted autoplay.
   - **Audio testimonials:** Convert `.m4a` → `.mp3` if needed. Keep short (≤60 sec excerpt) for inline player.
4. **Place under `assets/`:**
   - Per-program photos: `assets/programs/<slug>/`
   - Hero video / homepage: `assets/hero/`
   - Testimonials: `assets/testimonials/`
   - About: `assets/about/`
5. **Reference in the page** with appropriate alt text + lazy loading.
6. **Mark in this index** that the asset was pulled (move from "available" to "used" status if we want fine-grained tracking).

For a first pass we can use ImageMagick or any local image editor. The repo has no asset pipeline yet; we can add one (e.g., a `scripts/optimize-image.sh`) when the per-page work warrants it.
